# Personal Profile (Neon Identity)

这不是“把设计理念写在页面上”的展示页，而是一套可直接使用的个人主页：用强气质的视觉语言承载清晰的信息结构，让访问者在第一屏获得记忆点，同时能快速找到作品与联系方式。

## 设计理念

1. 第一屏要“有冲击”，但不牺牲可读性  
   背景采用霓虹氛围与网格透视的组合，文字层级保持克制：标题负责记忆点，副标题负责信息密度，CTA 负责行动路径。

2. 氛围是系统，不是一次性装饰  
   通过主题切换（霓虹/脉冲/冰蓝）在不改变信息结构的前提下切换情绪，并用 `localStorage` 记住偏好，让“视觉风格”像功能一样稳定存在。

3. 互动要轻，但要有反馈  
   互动的目标不是炫技，而是让页面“活起来”：终端打字、今日信号、扫描彩蛋、按钮磁吸等都属于低成本高反馈的微交互。

4. 信息架构优先于视觉解释  
   首页只放“你是谁 + 你做什么 + 去哪里看作品 + 如何联系你”，把详细故事放到 About，把项目放到 Projects，把合作方式放到 Services，把收口入口放到 Contact。

## 页面结构

- `index.html`：个人主页入口（简介、Now、精选项目、技能栈、快速入口、邮箱复制、终端互动）
- `about.html`：个人介绍与方法论（手风琴 + 时间线）
- `projects.html`：项目矩阵（筛选 + 右侧预览面板）
- `services.html`：服务模块（套餐卡片 + 详情联动）
- `contact.html`：联系模块（表单演示 + 快捷填入）

## 趣味互动（无后端、直接打开可用）

- 主题切换：霓虹/脉冲/冰蓝（记忆用户偏好）
- 首页终端：进入页面自动打字，模拟“开机”
- 今日信号：按当天日期生成固定的“今日状态”
- 扫描彩蛋：进度条扫描 + 粒子爆发反馈
- 头像彩蛋：点击头像切换身份（并记忆）
- 邮箱复制：一键复制邮箱（带短暂状态提示）

这些逻辑集中在 `assets/main.js`，样式在 `assets/styles.css`。

## 视觉语言与动效策略

- 背景：渐变 + 光斑（orbs）+ 网格透视 + 扫描线质感
- 卡片：玻璃拟态（半透明面板 + 细边框 + 内阴影）
- 动效：尽量使用轻量的 CSS/JS 组合  
  例如粒子漂移、按钮磁吸、终端打字，避免复杂依赖，保证可维护性与加载体验。

## 如何替换成你的信息

优先替换这些字段（示例占位）：

- 邮箱：`index.html` / `contact.html` 中的 `hello@your-domain.com`
- 社媒：`contact.html` 中的 `@yourhandle`
- 文案：`index.html` 第一屏简介、`about.html` 的自我介绍与时间线、`projects.html` 的项目描述

如果你想换名字/头像字母：

- 首页标题：`index.html` 的 `NICK`
- 头像字母：`index.html` 的按钮 `[data-avatar-toggle]`

## 本地预览

这是纯静态站点：

1. 直接双击打开 `index.html` 即可预览。
2. 或用任意静态服务器（例如 VS Code Live Server）获得更稳定的相对路径体验。
